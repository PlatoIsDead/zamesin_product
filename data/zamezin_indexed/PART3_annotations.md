## Лекция 1 — DEMO AJTBD-интервью: B2C, последовательная работа (аренда квартиры)
**Источник:** Zamesin 4.md (начало)
**Ведущая:** Вика Панкевич
**Продолжительность:** ~95 мин

**Главная идея:**
Полное демо-интервью для изучения последовательной (а не частотной) работы. Бизнес-задача: найти гипотезы сегментов для расширения сервиса аренды квартир. Показывает ключевые отличия от интервью по частотной работе.

**Ключевые темы:**
- **Последовательная работа vs частотная:** у последовательной работы несколько итераций в жизни (переезд 1-й, 2-й, 3-й раз) — у каждой итерации может быть разный контекст и разный ожидаемый результат.
- **Выбор работы для изучения:** вопрос «какой из ожидаемых результатов был самым важным и сложным?» — именно он помогает выбрать правильную работу для детального изучения.
- **Типичная ошибка при изучении предыдущих работ:** можно получить предыдущую работу не из нашего графа, а из другого. Нужно явно привязывать вопрос к конкретной работе выше уровнем.
- **Разбор формулировки работы в шаблоне:** как переносить контекст, психологические особенности, триггер и активирующее знание в описание работы на доске FigJam.
- **Выводы по сегментам:** на основе графа работ разных респондентов → гипотезы сегментов, на которых можно расширить сервис. Сравнение с конкурентом (Яндекс) — у разных продуктов разный граф работ для одной BigJob.

**Концепции / инструменты:**
Последовательная работа, Навигация работы (итерации), Предыдущие работы (правильный способ изучения), Шаблон описания работы, Граф работ → гипотезы сегментов

## Лекция 2 — Типы и свойства работ
**Источник:** Zamesin 4 2.md
**Продолжительность:** ~46 мин

**Главная идея:**
Систематизация всех типов и свойств работ. Понимание типов напрямую влияет на продуктовые решения — у каждого типа свои механики работы с ним.

**Ключевые темы:**
- **Последовательные работы:** человек не может не выполнить все шаги в цепочке. Ценность и польза получаются в конце или ближе к концу последовательности. Следствие: нужно максимально быстро приводить к этой ценности, убивая лишние работы.
- **Частотные работы:** одна работа может быть одновременно последовательной и частотной. Чем выше частотность — тем важнее для бизнес-моделей на подписке/возвращаемости.
- **Вертикальные (вёральные) работы:** работа выполняется для/вместе с другим человеком. Такие работы интересны тем, что один пользователь может приводить других (виральность). Фигма, Google Docs — продукты, построенные вокруг вёральных работ.
- **Ориентационные работы:** цель — обновить consideration set и узнать, как выглядит граф работ. Все работы с глаголами «понять», «узнать», «найти», «выбрать», «разобраться» — ориентационные. Поиск, Perplexity, Google живут на ориентационных работах.
- **Налоговые работы:** незапланированные работы, возникающие когда решение облажалось. Человек испытывает раздражение. Чинить налоговые работы = мощная механика создания ценности.
- **Ложные работы:** фантазии про будущее («я похудею», «выучу английский»). Диагноз: человек говорит в будущем времени. Метод: спросить, делал ли что-то для этого в прошлом.
- **Как продавать непривычное решение через BigJob:** когда нашего продукта нет в consideration set — идём на уровень выше, находим работу за которую конкурируют другие решения, доказываем ценность для неё.

**Концепции / инструменты:**
Типы работ (последовательная / частотная / вертикальная / ориентационная / налоговая / ложная), Consideration set (углублённо), Активирующее знание, Продажа непривычных решений через BigJob

## Лекция 3 — Воркшоп #4: Как запустить новый продукт и найти PMF
**Источник:** Zamesin #4- Как запустить новый продукт и найти для него PMF.md
**Продолжительность:** ~145 мин

**Главная идея:**
Первый алгоритмический воркшоп тренинга. Главный тезис: цель запуска — не запустить продукт, а убить его как можно быстрее и дешевле, найдя подтверждение (или опровержение) ключевых рискованных предположений.

**Ключевые темы:**
- **Почему «убивай продукт» — правильная цель:** большинство команд тащат дохлую лошадь из-за психологических ошибок. Пивот на раннем этапе стоит копейки. Пивот после года разработки — катастрофа.
- **5 рискованных предположений при запуске:**
  1. Есть сегмент людей, объединённых работой
  2. Мы можем создать значимо более ценное решение для этого сегмента
  3. У нас есть ресурсы и компетенции
  4. Мы знаем, как создать спрос
  5. Экономика сходится
- **Инструмент RAT (Riskiest Assumption Test):** для каждого рискованного предположения — минимальный эксперимент для его проверки. Как ранжировать риски по вероятности и критичности.
- **Алгоритм запуска нового продукта:** выбор рынка → поиск и выбор сегмента → создание и валидация ценности → тестирование привлечения → масштабирование.
- **Психологические ошибки:** confirmation bias, loss aversion при пивоте, авторитет основателя блокирует честную оценку.
- **Кейс: онлайн-образование (СПО и бакалавриат):** как выбирали рынок, как проверяли гипотезы сегментов, почему не пошли в очевидный сегмент.
- **Количество интервью для поиска сегмента:** B2C — 40-60, B2B — 15-50. Критерий полноты: за последние 10 интервью ничего принципиально нового не услышали.

**Концепции / инструменты:**
Алгоритм запуска нового продукта, RAT (Riskiest Assumption Test), 5 рискованных предположений, Психологические ошибки при запуске, Пивот

## Лекция 4 — Количественная валидация JTBD-исследования
**Источник:** Zamesin Количественная валидация JTBD-исследования.md
**Продолжительность:** ~180 мин

**Главная идея:**
Как после качественного исследования (интервью) перейти к количественному — чтобы измерить размер сегментов, подтвердить приоритет работ и принять решение о фокусе с числами в руках.

**Ключевые темы:**
- **Зачем нужна количественная валидация:** качество даёт полноту картины, количество даёт точность и уверенность. Только качество — «мне кажется, сегмент большой». Качество + количество — «сегмент составляет X% рынка».
- **Структура количественного опроса:** вопросы на квалификацию → вопросы на частотность работ → вопросы на важность → вопросы на удовлетворённость текущим решением.
- **Метрики для каждой работы:**
  - Доля сегмента (% людей, у которых есть эта ситуация)
  - Частота (сколько раз в месяц возникает)
  - Важность (регрессионный коэффициент, шкала 1-10)
  - Неудовлетворённость (10 минус удовлетворённость, шкала 1-10)
- **Итоговый скор привлекательности работы:** доля × частота × важность × неудовлетворённость. Чем выше — тем интереснее сегмент для продукта.
- **Визуализация на карте:** оси «важность» и «неудовлетворённость» → quadrant-анализ, где правый верхний угол = максимально привлекательные сегменты.
- **Кейс: посудомоечная машина** — разбор реального опроса с расчётом скора по разным работам (мытьё посуды повседневно, детская посуда, показать себя продвинутым пользователем и др.).
- **Ограничения количественного подхода:** нельзя делать количественные выводы после качественного исследования. Нельзя начинать с количества, минуя качество.

**Концепции / инструменты:**
Количественный опрос по JTBD, Скор привлекательности сегмента (доля × частота × важность × неудовлетворённость), Карта сегментов, Качество → количество как последовательность

## Лекция 5 — Как развивать существующий продукт
**Источник:** Zamesin Как развивать существующий продукт.md
**Продолжительность:** ~60 мин

**Главная идея:**
Как думать о развитии продукта, у которого уже есть клиенты. Два режима: локальный оптимум (оптимизируем то, что есть) и глобальный оптимум (ищем новые сегменты и работы). Как их совмещать.

**Ключевые темы:**
- **AB(CD)-сегментация (Сегментация Шона Эллиса):** делим текущих клиентов по маржинальности и удовлетворённости. A — максимально довольные и маржинальные. B — довольные, но менее маржинальные. C — недовольные, «тащим» с ними. D — убыточные. Кейс Kitkин: продукт работал идеально для халявщиков, а не для платящих.
- **Локальный vs глобальный оптимум:**
  - Локальный: работаем с теми, кто уже пришёл, чиним их проблемы, растим ценность. Это делегируется менее компетентным сотрудникам.
  - Глобальный: ищем сегменты за пределами текущей базы, выходим на работы выше уровнем, меняем бизнес-модель. Это стратегические решения.
- **Шаблонные исследования для развития продукта:**
  - Для роста привлечения: интервью с теми, кто не купил и не знает о продукте
  - Для роста ценности: интервью с AB-сегментами, что мешает давать больше ценности
  - Для снижения оттока: интервью с теми, кто ушёл — какая работа не была выполнена
- **Чем развитие существующего продукта отличается от запуска нового:** при запуске — весь фокус на поиске сегмента и первых продажах. При развитии — задачи масштабирования, эффективности, стратегии.
- **Не все клиенты одинаково полезны:** вы можете сливать деньги на привлечение и поддержку C/D-сегмента, который фундаментально не подходит под вашу бизнес-модель.

**Концепции / инструменты:**
AB(CD)-сегментация, Сегментация Шона Эллиса, Локальный vs глобальный оптимум, Шаблонные исследования, Развитие vs запуск

## Лекция 6 — Воркшоп #5: Как найти и выбрать сегмент для нового продукта
**Источник:** Zamesin #5- Как найти и выбрать сегмент для нового продукта.md
**Продолжительность:** ~160 мин

**Главная идея:**
Кульминационный воркшоп блока. Алгоритм поиска сегмента — самая главная гипотеза при запуске нового продукта. Четыре силы прогресса, барьеры, привычки и практический алгоритм итерационного поиска сегментов.

**Ключевые темы:**
- **Почему сегмент — главная гипотеза:** без сегмента невозможно создать ценность, привлечь клиентов и построить экономику. Каким бы прекрасным ни был офис и бренд — без сегмента это смертный приговор.
- **Типичные ошибки при поиске сегмента:**
  - Выбирать сегмент только по проблемам (игнорировать барьеры и привычки)
  - Не учитывать размер рынка и экономическую привлекательность
  - Выбрать «очевидный» сегмент без проверки
  - Не различать качественную полноту и количественную точность
- **4 силы прогресса (Forces of Progress):**
  1. Push: проблемы с текущим решением толкают к смене
  2. Pull: привлекательность нового решения тянет к нему
  3. Anxiety: тревога о том, что новое решение может не сработать — тормозит
  4. Habit: сила привычки к текущему решению — тормозит
  Хороший сегмент: сильный Push + сильный Pull + слабые Anxiety + слабая Habit.
- **Как оценивать сегменты по экономической привлекательности:** бюджет на выполнение работы, частотность, бизнес-модель (транзакция / подписка / комиссия), размер рынка.
- **Барьеры и привычки:** барьеры можно снижать (изменить коммуникацию, дать гарантии, снизить цену входа). Привычку сложнее — ищите сегменты, где привычка к текущему решению слабее.
- **Алгоритм итерационного поиска сегментов:**
  1. Гипотеза широкой работы → рекрутируем по этому критерию
  2. Итерация 6-8 интервью → выделяем сегменты → отбрасываем неинтересные → усредняем похожие
  3. Следующая итерация: углубляемся в интересные сегменты
  4. Повторяем до полноты (последние 10 интервью ничего нового)
- **Факторы выбора финального сегмента:** экономическая привлекательность + возможность создать значимо лучшее решение + ресурсы и компетенции.
- **Количество интервью:** B2C ~40-60, B2B ~25. Критерий полноты: 10 интервью подряд без нового.

**Концепции / инструменты:**
4 силы прогресса (Forces of Progress), Барьеры и привычки, Алгоритм итерационного поиска сегментов, Критерии выбора сегмента, «Сегмент не сегмент»

## Лекция 7 — Как рекрутировать респондентов на международных рынках
**Источник:** Zamesin Как рекрутировать респондентов на международных рынках.md
**Ведущий:** Вадим Гласков (агентство исследований на глобальном рынке)
**Продолжительность:** ~43 мин

**Главная идея:**
Практическое руководство по привлечению респондентов на зарубежных рынках. Кардинальных отличий от российского рынка нет — есть страновые нюансы и набор дополнительных инструментов.

**Ключевые темы:**
- **Главный инсайт:** люди везде очень похожи. Каждая локация — отдельное сообщество со своими нюансами (разные штаты США ≠ одна аудитория, Европа — разные страны, Индия — разные штаты и языки).
- **Страна как квалифицирующий признак сегмента:** сегмент может объединять людей из нескольких стран (пример: страховка + спортивный образ жизни работает в части штатов США и Британии).
- **Инструменты привлечения:**
  - **Панели респондентов** (userinterviews.com, respondent.io): самый быстрый способ (интервью через 2-3 дня), но дорогой ($70-140 за B2C, $100-150 за B2B). Есть возможность загрузить своих клиентов.
  - **LinkedIn (Sales Navigator):** лучший инструмент для B2B. Долго (до месяца до первых интервью), требует прокачанного аккаунта и умения писать цепочки холодных писем.
  - **Холодные письма:** ключевые элементы — ванлайнер (персонализация, показывает что не бот), win-win для респондента, чёткий CTA. Тестировать разные варианты и мерить конверсию.
  - **Платить или не платить:** в США — платить принято ($30-50 B2C, $100-150 B2B). В Азии — оплата воспринимается как скам. Топ-менеджеров деньгами не заинтересуешь — нужна ценность продукта.
- **Инструмент iTalki:** для дешёвых B2C-интервью с носителями языка целевого рынка ($10-20 за сессию).
- **Частые ошибки:** думать, что все пользуются WhatsApp и Zoom, ждать, что B2C-аудитория ведёт календарь. Нужно подстраиваться под инструменты респондента.
- **Рекрутмент-фабрика:** внутренний процесс команды, который на входе получает профиль нужного респондента, на выходе даёт забронированный слот в календаре.

**Концепции / инструменты:**
Панели респондентов, LinkedIn/Sales Navigator, Холодные письма + ванлайнер, Скринер с контрольными вопросами, Рекрутмент-фабрика

Вторая часть лекции Вадима Гласкова — про то, как проводить сами интервью с представителями других культур. Технические особенности и культурные нюансы, которые влияют на качество данных.

## Лекция 9 — Самостоятельная практика #3: Выделять сегменты из исследования
**Источник:** Zamesin Самостоятельная практика #3.md
**Продолжительность:** ~180 мин

**Главная идея:**
Практическое занятие: что делать с результатами интервью. Упражнение «сегмент / не сегмент» — как из 8 транскриптов выделить экономически интересные сегменты, отбросить неинтересные и усреднить похожие.

**Ключевые темы:**
- **Что делать после интервью:** каждый транскрипт → работа конкретного человека → оценка привлекательности → решение «интересный сегмент или нет».
- **Упражнение «сегмент / не сегмент»:** для каждого респондента — является ли его работа потенциально интересным сегментом для продукта? Критерии: экономическая привлекательность, возможность создать ценность.
- **Усреднение сегментов по похожести работ:** когда у нескольких респондентов похожий «когда-хочу-чтобы» — это один сегмент. Пишем усреднённую работу.
- **Кейс B2B (платформа для онлайн-курсов):** руководитель создаёт курс и отправляет сотрудникам. 8 интервью с разными типами руководителей → выделяем сегменты.
- **Кейс B2C (тот же продукт, другая аудитория):** альтернативный вариант практики.
- **Цель практики:** не научиться делать исследование под ключ, а научиться принимать решения на основе небольшого набора интервью.

**Концепции / инструменты:**
Анализ интервью, Упражнение «сегмент / не сегмент», Усреднение сегментов по похожести работ, Рабочая тетрадь
