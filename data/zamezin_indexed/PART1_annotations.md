## Лекция 1 — Воркшоп из 13-й версии: Новые элементы методологии
**Источник:** Zamesin.md
**Продолжительность:** ~160 мин

**Главная идея:**
Введение в обновлённую версию методологии Advanced JTBD. Замезин впервые публично представляет новые концепции, которые «закрыли дырки» в старой версии.

**Ключевые темы:**
- **Ориентационные работы** — новый тип работ. Когда человек выполняет работу впервые, он не знает граф — у него есть серия работ типа «понять», «узнать», «сориентироваться». Именно на этом живут Google, Яндекс, Perplexity.
- **Consideration set** — список всех известных человеку решений для выполнения работы. Формируется через ориентационные работы.
- **Активирующее знание** — знание о более ценном решении, которое может запустить смену текущего решения. Как продавать непривычные продукты: выходить на Big Job, а не на фичи.
- **Core Job vs Big Job** — Core Job: самая высокоуровневая работа, которую решение выполняет полностью. Big Job: работа выше уровнем, которую решение выполняет лишь частично. Правильное определение этих уровней — стратегическое решение.
- **Критическая последовательность работ** — набор Small Jobs, которые человек обязан выполнить, чтобы получить ценность от Big Job. Если продукт не закрывает хотя бы одну из них — клиент разочарован.
- **Критерии ожидаемого результата** — что именно человек ожидает от выполнения каждой работы. Ожидания первичны: один и тот же продукт при завышенных ожиданиях даёт разочарование, при заниженных — восторг.

**Концепции / инструменты:**
Ориентационная работа, Consideration set, Активирующее знание, Core Job, Big Job, Критическая последовательность работ, Критерии ожидаемого результата

## Лекция 2 — Воркшоп #1: Введение в Advanced Jobs To Be Done
**Источник:** Zamesin 2.md (основная часть)
**Продолжительность:** ~160 мин

**Главная идея:**
Водный воркшоп для новых участников тренинга. Главная задача — совершить переход от мышления фичами к мышлению работами клиента. Работа клиента — первопричина любого продуктового решения.

**Ключевые темы:**
- **Формат и структура тренинга:** 7 воркшопов, практики, домашние задания. Первые 3 воркшопа — база, с 4-го — бизнес-алгоритмы.
- **Главный тезис:** «Решение любой бизнес-задачи в зоне компетенции продукта должно начинаться с понимания работ клиента. Работа — первая причина».
- **Что такое работа:** Человек в точке A хочет прийти в точку B. Контекст + желаемый результат = работа.
- **Демо AJTBD-интервью:** Живой разбор интервью с участником (продукт — доставка еды). Показывает, как за 2-3 вопроса выявить несколько Core Jobs одного решения.
- **Сегментация по работам:** Первое знакомство с идеей сегментировать людей не по демографии, а по работам.
- **Маркетинговая коммуникация:** Тексты и рекламу строить из знания работ + ценностей + барьеров — это алгоритм, а не творчество.
- **Почему нельзя пропускать базу:** Интуитивное понимание продукта у большинства специалистов неполно. Без перезаписи база методология не ляжет.

**Концепции / инструменты:**
Job-to-be-done, Граф работ (введение), AJTBD-интервью (демо), Сегментация по работам (введение)

## Лекция 3 — Воркшоп #1: Продолжение (граф работ, сегментация)
**Источник:** Zamesin 2 2.md
**Продолжительность:** ~50 мин

**Главная идея:**
Завершение первого воркшопа. Граф работ как главная единица анализа методологии. Первые шаги в сегментации по работам.

**Ключевые темы:**
- **Граф работ** — не одна работа, а взаимосвязанная сеть работ разных уровней. Главная единица анализа AJTBD. Методология дала единицу «работа», AJTBD расширяет её до графа.
- **Почему граф важнее одной работы:** Зная только одну работу, вы не понимаете ни критическую последовательность, ни Big Job, ни ценность, ни как привлекать клиентов.
- **Практика мышления работами:** Упражнения в зале — участники пишут гипотезы Core Jobs и Big Jobs для своих продуктов.
- **Как исследование меняет мышление:** После проведения первого AJTBD-интервью мозг начинает автоматически разделять работу и решение — это «магический» момент обучения.
- **Ответы на вопросы:** Как выбрать правильную выборку для интервью. Что первично — гипотезная сегментация или интервью. Ответ: на 5-м воркшопе.

**Концепции / инструменты:**
Граф работ (углублённо), Core Job / Big Job (практика), AJTBD-интервью (первая практика)

## Лекция 4 — Воркшоп #2: Научиться работать с главными инструментами AJTBD
**Источник:** Zamesin #2- Научиться работать с главными инструментами AJTBD.md
**Продолжительность:** ~180 мин

**Главная идея:**
Центральный воркшоп базового модуля. Главный сдвиг фокуса: не продукт для всех, а продукт для сегмента по работам. Сегментация — ключевое стратегическое решение для любого продукта.

**Ключевые темы:**
- **Сегментация по работам** — почему демография не работает. Работа — первый и обязательный критерий сегментации. Без работы нельзя создавать ценность, привлекать, конкурировать.
- **Типичные ошибки сегментации:** Берут только тех, кто заполнил форму; смотрят только на тех, кто платит; игнорируют тех, кто ушёл к конкуренту.
- **Core Job и Big Job (углублённо):** Практика с реальными продуктами участников. Для каждого продукта — определить уровень Core Job и Big Job.
- **Критическая последовательность работ (углублённо):** Кейс Figma — как компания последовательно закрывала всё больше Small Jobs дизайнеров внутри своего продукта.
- **Критерии ожидаемого результата (КОР):** Что человек ожидает на выходе каждой работы. КОР — инструмент, который позволяет находить разрывы между ожиданием и реальностью продукта.
- **Масштабирование внутри сегмента:** Алгоритм: выбрал сегмент → изучил подсегменты → создал ценность → привлёк → повторил.
- **Ответ на вопрос «зачем сегментировать»:** Те же ресурсы → фокус → значимая ценность → выделение среди конкурентов → прибыль → финансирование роста.

**Концепции / инструменты:**
Сегментация по работам, Core Job / Big Job (применение), Критическая последовательность работ, Критерии ожидаемого результата (КОР), Масштабирование сегментов

## Лекция 5 — DEMO AJTBD-интервью (B2C, частотная работа)
**Источник:** Zamesin DEMO-интервью.md
**Продолжительность:** ~56 мин

**Главная идея:**
Полноценное демо-интервью с реальным участником тренинга. Показывает, как шаг за шагом применять гайд AJTBD-интервью для изучения частотной работы в B2C-сегменте.

**Ключевые темы:**
- **Реальное интервью в прямом эфире:** Интервьюер (Вика Панкевич) и респондент — участница тренинга, которая купила курс для решения задач своего фотобизнеса.
- **Шаги интервью:** Навигация работы → выбор самой частотной → контекст и триггер → ожидаемый результат → предыдущие работы → рассматриваемые альтернативы → удовлетворённость → барьеры.
- **Что узнали из интервью:** Core Job клиентки — «понять, что нужно людям, чтобы создавать то, что нужно, а не то, что кажется классным». Big Job — «быть уверенным предпринимателем».
- **Что ценного для продукта:** Клиентка ценит вечный доступ, возможность возвращаться к материалам. Барьер при покупке — высокая рабочая нагрузка (конец года). Рассматривала: книгу, фриланс-исследователя.
- **Разбор после интервью:** Замезин комментирует, что было сделано правильно, где можно было копнуть глубже.

**Концепции / инструменты:**
AJTBD-гайд 3.3 (практика), Навигация работы, Ожидаемый результат, Consideration set, Триггер, Барьеры, КОР

## Лекция 6 — Как проводить глубинные интервью
**Источник:** Zamesin Как проводить глубинные интервью.md
**Продолжительность:** ~38 мин

**Главная идея:**
Базовая лекция от Вики Панкевич для тех, кто никогда не проводил интервью. Практические советы по подготовке, проведению и анализу — без лишней теории.

**Ключевые темы:**
- **Подготовка:** Нельзя идти без дизайна исследования. Сначала: цели, сегменты, вопросы — потом интервью. Длительность: 45-60 минут. Меньше 30 — не хватит на эмоциональный контакт.
- **Установка эмоционального контакта:** Начинать с простых, не угрожающих вопросов (хобби, где живёт). Смотреть в камеру. Показывать заинтересованность.
- **Какие вопросы задавать:** Открытые, следовательские («расскажи подробнее»), про конкретные ситуации из прошлого. Не гипотетические («а что бы ты сделал если...»).
- **Какие не задавать:** Наводящие, закрытые, про будущее поведение — человек не знает, что будет делать.
- **Анализ интервью:** Сразу после — зафиксировать ключевые инсайты. Не тратить время на расшифровку всего — только значимые цитаты.
- **Присутствие коллег:** Как организовать наблюдателей, чтобы они не мешали эмоциональному контакту.

**Концепции / инструменты:**
Дизайн исследования, Эмоциональный контакт, Открытые вопросы, Анализ интервью

## Лекция 7 — Что такое UX-тесты
**Источник:** Zamesin Что такое UX-тесты.md
**Продолжительность:** ~18 мин

**Главная идея:**
Короткая лекция о том, что такое UX-тестирование, чем оно отличается от usability-тестирования, и как применять итерационный подход RITE для быстрого улучшения интерфейса.

**Ключевые темы:**
- **UX-тест:** Модерируемая сессия, где пользователь выполняет задачи с прототипом или продуктом. Цель — найти проблемы понимания и использования.
- **RITE-подход** (Rapid Iterative Testing and Evaluation): Берёшь 4 респондента → находишь проблемы → чинишь → берёшь следующих 4. Цель: ноль новых проблем на последней итерации.
- **Отличие от usability-тестирования:** Usability — формализованный инструмент для измерения времени и точности (нужен большой бюджет). UX-тест — быстрый, дешёвый, качественный формат для итеративного улучшения.
- **Когда применять:** На любых прототипах, включая Zoom + мобайл, мейлы, тексты, скрипты, офлайн-продукты.
- **Как начинать тест:** Ставить клиента в контекст — объяснять ситуацию, в которой он «пришёл» к продукту.

**Концепции / инструменты:**
UX-тест, RITE-подход, Usability testing (отличие), Прототипирование

## Лекция 8 — Как устанавливать эмоциональный контакт
**Источник:** Zamesin Как устанавливать эмоциональный контакт.md
**Продолжительность:** ~100 мин

**Главная идея:**
Развёрнутая лекция-практикум об эмоциональном контакте как ключевом навыке интервьюера. Без эмоционального контакта респондент не раскроется — даже правильные вопросы не дадут инсайтов.

**Ключевые темы:**
- **Почему это критично:** Если интервью длится 45-60 минут, и половина уходит на установку контакта, у вас остаётся 20-25 минут на содержательную часть. Без контакта — отписки вместо инсайтов.
- **Техники установки контакта:** Small talk перед интервью, открытые вопросы про жизнь, демонстрация безопасности среды, зрительный контакт через камеру.
- **Работа с разными типами респондентов:** Замкнутые, разговорчивые, уводящие в сторону — для каждого типа свои приёмы.
- **Ошибки интервьюера:** Перебивать, выражать согласие/несогласие, торопиться к следующему вопросу, демонстрировать удивление — всё это разрушает контакт.
- **Практика:** Разбор конкретных реплик — как перефразировать, чтобы не потерять контакт и продолжить копать.
- **Связь с качеством данных:** Чем глубже контакт — тем честнее ответы, тем ценнее инсайты для продукта.

**Концепции / инструменты:**
Эмоциональный контакт, Активное слушание, Типы респондентов, Ошибки интервьюера

## Лекция 9 — Самостоятельная практика #2: Научиться изучать граф работ
**Источник:** Zamesin Самостоятельная практика #2.md
**Продолжительность:** ~180 мин

**Главная идея:**
Практическое занятие в парах, где участники учатся изучать граф работ для реального продукта. Цель: научиться находить Core Jobs, предыдущие/следующие работы и Big Jobs через мини-интервью.

**Ключевые темы:**
- **Формат практики:** Работа в парах — один в роли интервьюера (продакт-менеджер), другой в роли пользователя продукта. Продукт выбирает респондент.
- **Задача интервьюера:** Найти гипотезы роста ценности и способы привлечения клиентов через изучение графа работ.
- **Шаги практики:**
  1. Выбрать продукт от лица которого работаешь
  2. Провести мини-AJTBD-интервью
  3. Определить Core Job, предыдущие и следующие работы, Big Job
  4. Записать в рабочую тетрадь
- **Разбор результатов:** Замезин в прямом эфире разбирает ответы участников — корректирует определение уровней работ, указывает на типичные ошибки.
- **Типичная ошибка:** Путать уровни Core Job — писать слишком абстрактно (Big Job) или слишком конкретно (Small Job).

**Концепции / инструменты:**
Граф работ (практика), Core Job / Big Job (применение), AJTBD-интервью (мини-формат), Рабочая тетрадь
